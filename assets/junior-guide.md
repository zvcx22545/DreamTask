# ยินดีต้อนรับสู่ Project Dream (คู่มือสำหรับ Junior Developer)

Project Dream คือแอปพลิเคชันสำหรับจัดการงานร่วมกันในทีม ถ้าน้องๆ เคยใช้ Notion หรือ Trello มาก่อน จะคุ้นเคยกับแอปพลิเคชันนี้ได้ไม่ยากเลย

คู่มือฉบับนี้จะช่วยให้น้องๆ เข้าใจว่าโปรเจกต์นี้สร้างขึ้นมายังไง, วิธีเซ็ตอัปรันบนเครื่องตัวเอง ไปจนถึงไกด์ไลน์ว่าจะแก้โค้ดหรือเพิ่มฟีเจอร์แรกได้อย่างไรบ้าง!

## 1. Mental Model แจ่มๆ (ภาพรวมในหัว)

- **"ให้มองว่ามันคือ Notion clone สำหรับทำงานแบบทีม"** – ยูสเซอร์สามารถสร้างทาสก์ (Task) และพิมพ์เนื้อหาลงไปในนั้นด้วย Editor แบบยืดหยุ่น (TipTap) แล้วทำงานแชร์กันได้
- **Core Stack (เทคโนโลยีหลัก):**
  - **Frontend (หน้าบ้าน):** ถนัดรันบน Next.js (React), ตกแต่งด้วย Tailwind CSS, จัดการข้อมูลระหว่างแผงงานด้วย Zustand
  - **Backend (หลังบ้าน):** เป็น Express.js, ใช้ Prisma (เป็นตัวช่วยให้เราคุยกับฐานข้อมูลได้ง่ายๆ ไม่ต้องเขียน SQL ดิบ)
  - **Infrastructure (ระบบฐาน):**
    - **PostgreSQL:** ฐานข้อมูลหลักที่ใช้เก็บข้อมูลจริงจัง *(ช่วงแรกยังไม่ต้องไปกังวลเรื่อง Docker หรอก แค่รู้ว่ามันช่วยเสกฐานข้อมูลขึ้นมาเปิดหลังบ้านให้เราผ่านระบบจำลอง)*
    - **Redis:** คลังเก็บข้อมูลของชั่วคราวแบบ "เร็วทะลุนรก" ใช้จัดการตัวรอคิว (Queue) เบื้องหลัง
    - **Meilisearch:** เครื่องมือค้นหาแยกต่างหากที่ทำมาเพื่อค้นหาคำในโปรเจกต์ได้แบบปรู๊ดปร๊าดโดยไม่ต้องง้อ ฐานข้อมูลหลัก

## 2. เริ่มต้นรันโปรเจกต์บนเครื่อง (Step-by-Step)

มาลองรันโปรเซสทั้งหมดขึ้นมาทำงานบนคอมของเรากันเถอะ:

1. **โหลดแพ็กเกจ (Install Dependencies):**  
   พิมพ์สิ่งนี้ลงไปใน Terminal
   ```bash
   pnpm install
   ```
2. **ปลุกระบบหลังบ้านให้ทำงาน (Database, Redis, Search):**  
   *(ต้องมั่นใจก่อนนะว่าเปิดโปรแกรม Docker Desktop อาศัยอยู่แล้ว)*
   ```bash
   docker compose up -d
   ```
   ขั้นตอนนี้จะช่วยเสก PostgreSQL, Redis, และ Meilisearch ขนาดย่อมๆ เตรียมรอไว้ให้
3. **สั่งรัน Server สำหรับพัฒนาโปรแกรม (API & Web):**
   ```bash
   pnpm dev
   ```
   *(คำสั่งนี้จะไปรันทั้ง Express API ที่อยู่บนพอร์ต `8080` และหน้าเว็บ Next.js ที่อยู่บนพอร์ต `8000` พร้อมๆ กันทีเดียวจบ)*
4. **เปิดเว็บเบราว์เซอร์ดูผลงาน:**  
   เปิดไปที่ [http://localhost:8000](http://localhost:8000) คุณจะเห็นหน้าต้อนรับ หรือ หน้าล็อกอินทันที!

## 3. "อยากแก้ตรงเนี้ย... ต้องไปแก้ที่ไหน"

เราเก็บโค้ดทั้งหมด (หน้าเว็บ + API) ไว้ในโฟลเดอร์เดียวที่เรียกว่าโครงสร้างแบบ "Monorepo" 

- **อยากเพิ่มหรือแก้หน้าเว็บใหม่ (Frontend):**  
  ให้เข้าไปค้นใน `apps/web/app/`  
  *(ตัวอย่าง: หน้ารายการงานจะอยู่ที่ `apps/web/app/(dashboard)/tasks/page.tsx`)*
- **อยากปรับปรุง หรือ เพิ่มเส้นทาง API ใหม่ (Backend):**  
  ให้เข้าไปค้นใน `apps/api/src/routes/`  
  *(ตัวอย่าง: โค้ดที่ดึงงานให้มาแสดงจะอยู่ใน `apps/api/src/routes/tasks.ts`)*
- **อยากแก้อะไรที่เกี่ยวกับโครงสร้างฐานข้อมูล (Database Schema):**  
  ให้ไปแก้ไขที่ไฟล์นี้ทีเดียว `packages/db/prisma/schema.prisma`

## 4. Coding Standards (มาตรฐานการเขียนโค้ด)

เมื่อต้องเขียนโค้ด ให้จำเรื่องกติกาง่ายๆ ดังนี้:
- **ตัวแปรต่างๆ ให้ใช้รูปแบบ `camelCase`** (`myTaskVar` ลากให้ถูก นะ ไม่เอาแบบนี้ `my_task_var`)
- **ใช้ `async/await` เสมอ** แทนการเขียน `.then()` สำหรับงานที่ต้องยืนรอข้อมูลขากลับ
- **พึ่งพา ESLint + Prettier:** ให้เปิดระบบจัด format โค้ดออโต้ตอนเซฟไว้เสมอ หาก VS Code ขึ้นเตือนขีดแดงยาวๆ ให้แก้ให้ผ่านก่อน

## 5. ตัวอย่างงานจริง: การเพิ่มฟิลด์ใหม่ (Task Example)

สมมติว่ารุ่นพี่บอกว่า เราจำเป็นต้องจดแต้ม "Story Points" ให้ได้ใน Task ทำตามขั้นตอนด้านล่างนี้ได้เลย:

1. **แก้ Schema ฐานข้อมูล:**  
   เปิด `packages/db/prisma/schema.prisma` ขึ้นมา แล้วเพิ่มบรรทัด `storyPoints Int?` ลงไปที่โมเดลของ `Task`
2. **อัปเดตไปที่ฐานข้อมูล (Migrate):**  
   รันคำสั่ง `pnpm db:push` เพื่อผลักคำสั่งใหม่เข้าไปอัปเดตคอลัมน์ในฐานข้อมูล (ในโหมด Dev)
3. **เอาไปใส่ใน API:**  
   เปิดเข้าไปที่ `apps/api/src/routes/tasks.ts` มองหาจุดที่มีการสั่ง "สร้าง/อัปเดต" และสั่งให้ Prisma คิวรีดึงฟิลด์ `storyPoints` เข้าเก็บไปด้วยได้แล้ว
4. **โยงกลับไปที่หน้าจอ UI:**  
   กระโดดไปส่วนของ Web (`apps/web/...`) หาฟอร์มสำหรับแก้ Task แล้วใส่ช่อง Input อันใหม่เพื่อให้กรอกและส่งค่าเข้าไป

## 6. ปัญหาสุดฮิตที่ร้อยทั้งร้อยต้องเจอ (Common Issues)

- **รันพอร์ตไม่ได้ (Port 8000 / 8080 อยู่มาไม่ได้):**  
  น่าจะมีอะไรค้างดึงแย่งพอร์ตไว้อยู่ สั่ง kill ทิ้งให้สาสม!
  ```bash
  npx kill-port 8000
  npx kill-port 8080
  ```
- **เปิด Docker ไม่ขึ้น หรือ ดาต้าเบสเอ๋อกิน (Database is acting weird):**  
  ลบทิ้งแล้วเริ่มใหม่แบบคนแมนๆ (คำเตือน: ข้อมูลเก่าที่ลองทำเทสต์บนเครื่องจะหายเรียบนะ!)
  ```bash
  docker compose down -v
  docker compose up -d
  pnpm db:push
  ```

## 7. ข้อมูลเผื่อแผ่ไว้ดูตอน Deployment (Optional)

ส่องดูสักนิดว่าเว็บเราไปอยู่ที่ไหนตอนที่เพื่อนเห็นกันสดๆ บนเน็ต:
- **Web (หน้าบ้าน Next.js):** เราจะ Deploy มันขึ้น **Vercel** เพื่อให้ได้ประสิทธิภาพในการแสดงผลแบบไวปรี๊ด 
- **API & Database (หลังบ้าน):** เอาคุมโดยแพลตฟอร์มเจ๋งๆ อย่าง **Railway** หรือ **Fly.io** ที่จัดการเรื่อง Docker คอนเทนเนอร์ และฐานข้อมูล PostgreSQL ออกมาได้อย่างสบายมือ
